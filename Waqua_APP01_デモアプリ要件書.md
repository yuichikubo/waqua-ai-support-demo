## Claude Code向け実装指示書

---

## 1. プロジェクト概要

### 1.1 目的
株式会社Waqua（浄水器・ウォーターサーバーのレンタル・販売・メンテナンス事業）向けに、問い合わせ対応の自動化アプリ（APP-01）のデモを構築する。LINE・HP問い合わせ・電話の3チャネルから届く問い合わせを、AIが自動分類・自動応答し、未解決案件のみ担当者へエスカレーションする業務フローを体感できるプロトタイプ。

### 1.2 提案書上のAPP-01フロー
```
① 受付（HP・LINE・電話の入口を統一的に受け付け）
② 分類（レンタル・部品・FAQ等を自動判定、振り分けルール標準化）
③ 応答生成（FAQ自動返答、個別判断が必要な案件のみ担当者接続）
④ 通知（未解決案件の集約、放置防止の通知フロー）
```

### 1.3 技術スタック
- **フレームワーク**: Next.js 14（App Router）
- **スタイリング**: Tailwind CSS
- **言語**: TypeScript
- **状態管理**: React Context または Zustand
- **データ**: JSONファイルベースのモックデータ（DBは不要）
- **AIシミュレーション**: タイピングアニメーション付きの定型応答（実際のAI API接続は不要）

### 3.3 問い合わせ詳細（`/inquiries/[id]`）

#### 左ペイン（60%幅）— AI対応チャット
参考サイトのLINE/Email画面と同様のチャット形式UI。

**ヘッダー部分:**
- 顧客名、チャネル（バッジ）、ステータス
- 「AI」「人間」切り替えトグルボタン（参考サイトと同じ青/オレンジのボタン）

**チャットエリア:**
- 顧客メッセージ: 左寄せ（白背景の吹き出し）
- AI応答: 右寄せ（緑背景の吹き出し）—タイピングアニメーション付き
- 人間応答: 右寄せ（青背景の吹き出し）
- 各メッセージに送信日時を表示
- 「AIが応答を生成中...」のローディングインジケータを表示してから応答表示（1〜2秒の遅延演出）

**入力エリア:**
- テキスト入力フィールド + 「送信」ボタン
- Ctrl+Enter で送信対応

#### 右ペイン（40%幅）— 案件情報サマリー
- **AI分類結果**: カテゴリ名 + 信頼度スコア（例: 「レンタル希望 92%」）
- **顧客情報**: 氏名、連絡先、過去の問い合わせ件数
- **推奨アクション**: AIが提案する次のステップ（例:「レンタルカタログの送付を推奨」）
- **対応履歴**: 過去のやり取りサマリー（時系列で3件程度）
- **ステータス変更ボタン**: 「対応完了にする」「担当者に引き継ぐ」

- 自動応答の有効/無効トグル
- チャネルごとの自動応答ON/OFF（LINE / HP / 電話）
- 応答遅延時間（秒数スライダー: 1〜10秒）
- FAQ一致閾値（%スライダー: 50〜100%）

**2. 通知設定**
- 未対応案件の通知間隔（セレクト: 5分 / 15分 / 30分 / 1時間）
- エスカレーション条件（AI信頼度XX%以下で自動エスカレーション）
- 通知先メールアドレス設定

**3. 分類設定**
- 分類カテゴリの一覧表示（追加・編集・削除可能）
- デフォルト: レンタル希望 / 部品請求 / 故障・修理 / 料金・契約 / 設置・配送 / その他

## 4. ステータス遷移ルール

### 4.1 問い合わせステータスの定義と遷移条件

```
新規 → AI対応中 → 対応待ち → 解決済み
  │        │          │
  │        │          └→ 解決済み（人間が対応完了）
  │        └→ 解決済み（AI自動応答で解決）
  └→ 対応待ち（AI信頼度が低く即エスカレーション）
```

| 現ステータス | 次ステータス | トリガー |
|------------|-----------|---------|
| 新規 | AI対応中 | AI分類処理開始時（自動、1秒後） |
| AI対応中 | 解決済み | AIが応答を返し、追加質問なし or 顧客がFAQ回答で満足 |
| AI対応中 | 対応待ち | AI信頼度が設定閾値以下 / 顧客が「人間と話したい」/ AI分類が「故障・修理」かつ緊急 |
| 対応待ち | 解決済み | 担当者が「対応完了にする」ボタンをクリック |
| 新規 | 対応待ち | 電話チャネルは常に人間対応のため即遷移 |

### 4.2 カテゴリ別AI推奨アクション

| 分類カテゴリ | AI推奨アクション | 自動対応可否 |
|------------|----------------|------------|
| レンタル希望 | レンタルカタログ・料金表の送付を推奨。法人の場合は営業担当への引継ぎを推奨 | 一次対応可 |
| 部品請求 | 部品型番の確認 → 在庫確認 → 注文フォーム案内 | 一次対応可 |
| 故障・修理 | 症状ヒアリング → 簡易トラブルシュート案内。改善しない場合はサービス担当へエスカレーション | 一次対応可、解決しない場合エスカレ |
| 料金・契約 | 料金プラン案内 / 請求明細の確認方法案内。解約は人間対応必須 | FAQ対応可。解約はエスカレ |
| 設置・配送 | 設置可能エリアの確認 → 設置日程の調整案内 | 一次対応可 |
| その他 | 内容をヒアリングし、適切な部署へ振り分け | エスカレーション |

## 5. AI演出の詳細仕様

### 5.1 自動分類の演出
問い合わせ受信時、以下の流れをUIで表現する:
1. 問い合わせ受信 → ステータス「新規」
2. 1秒後 → 「AI分析中...」スピナー表示
3. 2秒後 → 分類結果表示（カテゴリ + 信頼度%）→ ステータス「AI対応中」
4. 分類結果に基づきFAQ一致があれば → 自動応答案を表示

### 5.2 チャット応答の演出
1. ユーザーメッセージ受信
2. 「AIが応答を生成中...」+タイピングインジケータ（…のアニメーション）
3. 1.5〜2.5秒（ランダム遅延）後に応答テキストが1文字ずつ表示（タイピングエフェクト）
4. 応答完了後、「この回答は参考になりましたか？」のフィードバックボタン表示

### 5.3 FAQ自動生成の演出
FAQ管理画面の「FAQ自動生成」ボタン押下時:
1. 「過去の問い合わせを分析中...」プログレスバー表示
2. 3秒後 → 「3件のFAQ候補を生成しました」トースト
3. テーブルに新規3件が「下書き」ステータスで追加

## 8. 実装上の注意事項

1. **ルーティング**: Next.js App Routerを使用。全ページは `layout.tsx` でサイドバー＋ヘッダーを共有する
2. **レスポンシブ**: デスクトップ表示（1280px以上）を前提とする。モバイル対応は不要
3. **データ永続化**: 全データはクライアントサイドのstateで管理。ページリロードでリセットされてOK
4. **AI API接続**: 不要。全てハードコードされたダミーレスポンスで対応
5. **認証**: ログイン画面は不要（常にログイン済み前提）。ヘッダーにユーザー名表示のみ
6. **日本語**: UI全体を日本語で構築。コンポーネント名・変数名は英語
7. **アニメーション**: チャットのタイピングエフェクトと、AI分類時のスピナーを実装。過度なアニメーションは不要
8. **recharts**: ダッシュボードのグラフ描画に使用。`npm install recharts` が必要
9. **Lucide React**: アイコンに使用。`npm install lucide-react` が必要
